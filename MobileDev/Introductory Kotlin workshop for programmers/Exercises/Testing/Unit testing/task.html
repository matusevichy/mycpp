Unit testing

Let's turn things around! We have some code we need to cover with our tests.

The common structure of a test file looks like this: 

    
class Test {

    @Before
    fun setup() {
        // Everything you like to do before each test has run
    }

    @After
    fun teardown() {
          // Everything you like to do after each test has run
    }

    @Test
    fun a_should_be_equal_to_a() {
        val expected = "a"
        val actual = "a"

        assertEquals(expected, actual, "Should return equal")
    }

}

A test framework will call each function annotated with @Test. Each function tries to test an assertion.
