Explanation

Just like other OO programming languages, classes in Kotlin may have type parameters: 


    class Box<T>(val value: T)

Here T is the type parameter and the property 'value' has the type T. The actual type that T represents is given at implementation of the Box class.

In general, to create an instance of such a class, we need to provide the type arguments: 


    val box: Box<Int> = Box<Int>(1)

This creates a box that will have an Int in the value field.

But if the parameters can be inferred, e.g. from the constructor arguments or by some other means, one is allowed to omit the type arguments: 


    val box = Box(1) // 1 has type Int, so the compiler figures out that we are talking about Box<Int>

In the previous example the type parameter could be anything, but it is usually desired to narrow the type. For example: 


    class NumberBox<T:Number>(val contents:T)

In the above example, the box may only contain numbers: 


    val box1 = NumberBox<Int>(123) //good
    val box2 = NumberBox<String>("abc") //compiler error!

You can read more in the Kotlin reference.
