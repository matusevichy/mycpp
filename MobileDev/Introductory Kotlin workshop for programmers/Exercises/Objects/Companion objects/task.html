Companion Object

Object declaration inside a class can be marked with the companion keyword.

The nice thing about a Companion object is that the functions or values declared inside it will look like they are part of the class it accompanies.

    
      class SomeClass {

        companion object CompanionObject {
            fun doSomething(){
                println("something")
            }
        }
      }
      val x = SomeClass.doSomething()
      val actualCompanionObject = SomeClass.CompanionObject

We don't even have to name the companion object:

    
      class SomeClass {
        companion object {
            fun doSomething(){
                println("something")
            }
        }
      }
      val x = SomeClass.doSomething()
      val actualCompanionObject = SomeClass.Companion

Assignment

One use case of a companion object could be an implementation of the factory pattern.

The factory’s goal is to deliver a ready-to-use object to its client. The factory hides the complexity of creating and selecting the right object for the job.

Now it's time for you to create a companion object.

You know exactly what a plumbus is and how it’s used. Now we will create a PlumbusFactory! The factory will create simple Plumbuses, assign each one an id, and count how many have been made.

There currently is a Plumbus class with a private constructor (so no one create his own home-grown Plumbus). It is your job to add a companion object by the name PlumbusFactory. This factory has two things:

  1. A count property indicating how many Plumbuses have been created.
  2. A create() function, which increments the count and returns a fresh Plumbus with an id that is equivalent to the new count/
