Platform types

Any reference in Java may be null, which makes Kotlin's requirements of strict null-safety impractical for objects coming from Java. Types of Java declarations are treated specially in Kotlin and called platform types.

A platform type is indicated by IntelliJ with an exlamation mark (!)

When you receive a reference from a java call, you can choose to treat it as a nullable or non-nullable. When you assign the reference to a non-nullable variable, an assertion will take place.

For example: 


    val resultA = someJavaCall() //result is a platform type

    resultA.someMethod() //throws NullPointerException if resultA is null

    val resultB:String = someJavaCall() //will throw IllegalStateException if result is null
    val resultC:String? = someJavaCall() //result is considered a nullable reference
