Import clash

Importing classes with the same name is not allowed in Kotlin. One way we can work around this issue is by omitting the import statement clashing class. Problem with this approach is that the we have refer to the full path of the clashing class when declaring it.


   import some.long.packagename.a.Foobar

   val fooBar : FooBar = FooBar()
   val fooBarB : some.long.packagename.b.FooBar = some.long.packagename.b.FooBar()

Luckily there is a way to import multiple classes with the same name without causing compilation error. In case of ambiguity we can use the keyword as to locally rename the clashing entity:


   import some.long.packagename.a.Foobar
   import some.long.packagename.b.Foobar as FoobarB

   val fooBar : FooBar = FooBar()
   val fooBarB : FoobarB = FooBarB()
