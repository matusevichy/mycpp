Properties

Classes in Kotlin can have properties. These can be declared as mutable, using the var keyword or read-only using the val keyword. 


    class Animal(){
        val nrOfLegs:Int = 0
        var name = "Kermit"
    }

To use a property, you can simply refer it by name: 


    val myAnimal = Animal()
    myAnimal.name = "Dumbo"

For declaring properties and initializing them from the primary constructor, Kotlin has a concise syntax: 


    class Animal(size:Int, val nrOfLegs:Int){
        var name = "Kermit"
    }

In this example, the property 'nrOfLegs' is initialized through the constructor. The 'size' parameter of the constructor however, is not a property.

Almost forgot: everything in Kotlin is public by default! If you do not want to expose something use the private keyword. e.g.

class Animal(size:Int, private val nrOfLegs){ var name = "Kermit" }

Inheritance

To declare an explicit supertype, we place the type after a colon in the class header: 


    open class Animal(size: Int)

    class Dog(size: Int) : Animal(size) //initialize Animal, using size

If the derived class has a primary constructor, the base class should be initialized right there, using the arguments from the constructor.

Every class in Kotlin is final by default. This means they can't be inherited from. To allow other classes to inherit a given class, it should have the keyword open.

Functions

Classes can have functions. Functions in a class are called member functions. They can be accessed the same way as properties.

Overriding functions

To override an inherited function, two things need to be in place

  • it needs to be open in the base class
  • the inheriting class uses the override keyword.

    open class Animal(size: Int){
        open fun eat(){ ... }
    }

    class Dog(size: Int) : Animal(size){
        override fun eat(){ ... }
    }
