Lambdas

Lambda expressions are function literals. This means they are not declared, but passed immediately as an expression. An example of a function call with a lambda: 


    max( strings, { a, b -> a.length < b.length } )

In this example, the lambda is the second argument of the max function; the part that says:
{ a, b -> a.length < b.length }
This expression takes two arguments, a and b, which are inferred to be strings, and returns true if b is longer than a.

Syntax

A lambda expression is

  • always surrounded by curly braces,
  • parameter declarations in the full syntactic form go inside curly braces and have optional type annotations,
  • the body goes after an -> sign.
And if the inferred return type of the lambda is not Unit, the last (or possibly single) expression inside the lambda body is treated as the return value.

If your lambda has a single parameter, you can refer to it as it. E.g. 


    { a:String -> it.length }

If a lambda parameter is unused, you can use the _. For example, we can change the first example: 


     max( strings, { _, b -> b.length == 12} )

Note: this would be a terrible max function call.
