Higher Order Functions

Higher order functions (HOF), is an expensive way of saying

A function that has a function as a parameter

Since functions are first-class citizens in Kotlin, you can pass functions around as parameters.

Just as a regular parameter: to have a function as a parameter, you have to define its type. The type of a function is defined in the following form: 

    (A , B) -> C

This specifies the signature of the expected function. Between parentheses, you specify its parameter types (A and B). After the -> you define its return type (C).

Functions that have no parameters are specified using empty braces, e.g. () -> C.

Functions that have no return value are always specified with the type Unit, e.g. (Int, Int) -> Unit

An example of a function definition of a higher order function: 


    fun theHoff( kittWristwatch: (String)->Boolean ){
        println("Kitt, execute!")
        val isSuccess = kittWristwatch("IncreaseSizeOfShark")

        if( isSuccess ) {
            println("Thanks Kitt!")
        } else {
            println("xâ¸‘x")
        }
    }

So, theHoff is expected a function he calls 'kittWristwatch'. This function should expect a String and should return a Boolean.

A call to theHoff looks as follows 


    val brokenWristwatch:(String) -> Boolean = { command:String ->
        println("Sorry, I can't do that Michael")
        false
    }

    theHoff( brokenWristwatch )

Off course, this can be shortened by applying the lambda directly: 


    theHoff{ command ->
        println("Sorry, I can't do that Michael")
        false
    }

Notice the absence of braces. If the lambda is the only argument to that call, the parentheses can be omitted entirely.

You can read more about Kotlin HOF in the Kotlin reference.

Assignment

Now for the assignment we want you to write a Stopwatch object. The Stopwatch object has a method which will measure the execution time of a given function.

It has the following requirements, it:

  • is named "elapse"
  • will execute the given function. The given function should not expect any parameters nor have any output (=Unit)
  • will return a the duration of that function in ms as a Long

Use 'System.currentTimeMillis()' for getting the current time.
