Before starting the course

First, let's make sure the setup of your environment is ok.

Main method

This method is sometimes added so you can play around with the assignment without having to rely on test output.

Try to execute the main method by pressing on the green arrow to its left.

If the green arrow is not present

There can be various reasons why the green arrow is not showing:

  • you need to install the Kotlin plugin. If you do not know how, ask an instructor (or the person next to you).
But, if the Kotlin plugin is already present, then there is a high likely hood of there being a gradle issue.
  • Open the gradle tool window on the right
  • first press 'refresh'
  • if that goes well, then 'execute gradle task' > popup window > type 'clean build'
There will probably be an error message in one of the steps, which could help you on your way.

Check task - step one

This is used to validate your assignment. It will kick off various Tests and inform you about the result.

Click on the box-checkmark icon in the top of this Task Description. It should try to kick off the tests. However, in this case it should return Compilation failure!

Check task - placeholder

Some assignments have placeholders, which indicate what to fill in where in the task.

To make the currentcheck work, you have to fill in the placeholder in the task. Type "TestMe" (without the quotes) in the placeholder, and run the 'check task' again.

When running "check task", it should now report Congratulations!. This means the placeholder was filled in correctly.
